// Commands
All actions available via button or CLI
Scan & Scope All Issues
Stage 1 — Devin analyzes every new issue and produces a scoping report. No code written.
python process_issues.py --scan
Scope Single Issue
Run Stage 1 on one specific issue by number.
python process_issues.py --issue <number>
Approve Issue
Stage 2 — dispatch a fix session to Devin after reviewing its scoping report.
python process_issues.py --approve <number> --mode <mode>
Rollback Issue
Close the open PR and reopen the issue for re-triage or manual handling.
python process_issues.py --rollback <number>
Dry Run (Preview)
Preview what IssueFlow would do without dispatching any Devin sessions.
python process_issues.py --scan --dry-run
Start API Server
Run the Flask backend so this dashboard can trigger commands directly.
python process_issues.py --serve
